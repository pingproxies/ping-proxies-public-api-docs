# Purchasing via API

The Ping Proxies API enables complete programmatic control over the purchasing and management of proxy services. This guide covers how to browse available products, generate quotes, create checkouts, and manage ongoing services.

## Overview of the Purchasing Process

<img className="block dark:hidden" src="/images/purchase-flow-light.svg" alt="API Purchase Flow" />
<img className="hidden dark:block" src="/images/purchase-flow-dark.svg" alt="API Purchase Flow" />

The typical purchasing flow involves these steps:

1. **Browse available products** using the catalog endpoint
2. **Generate a quote** to check pricing and availability
3. **Create a checkout** to complete the purchase
4. **Manage the service** after purchase

## Browsing Available Products

Start by retrieving the available products through the catalog endpoint:

```bash
curl --request GET \
  --url 'https://api.pingproxies.com/1.0/public/user/checkout/catalog' \
```

## Generating a Quote

Before completing a purchase, generate a quote to check pricing, availability, and any applicable discounts:

```bash
curl --request POST \
  --url 'https://api.pingproxies.com/1.0/public/user/checkout/quote' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Public-Key: your_public_key' \
  --header 'X-API-Private-Key: your_private_key' \
  --data '{
    "product_code": "isp_us",
    "quantity": 5,
    "cycle_interval": "month",
    "cycle_interval_count": 1,
    "promotional_code": "WELCOME10"
  }'
```

```bash
  --header 'X-API-Private-Key: your_private_key' \

    "product_code": "isp_us",

    "cycle_interval_count": 1,
    "promotional_code": "WELCOME10",
    "service_fulfillment_filter": {
```

### Payment Processing

The checkout response will provide one of two outcomes:

1. **Fully paid by credit**: If your account has sufficient credit balance to cover the entire purchase, the service will be provisioned immediately:
   ```json
   {
     "data": {
       "invoice_is_paid": true,
       "invoice_url": null,
       "service_id": "API-1234-5678"
     },
     "message": "Successfully created checkout and paid with credit. Your service will be provisioned shortly."
   }
   ```

2. **Requires additional payment**: If your credit balance doesn't cover the entire purchase, you'll receive an `invoice_url` where the payment can be completed:
   ```json
   {
     "data": {
       "invoice_is_paid": false,
       "invoice_url": "https://invoice.stripe.com/i/acct_1H7dAB2BUlqim5l/test_YWNjdF8xSGIzUEFCMkJabsEyzPS3Aag",
       "service_id": "API-1234-5678"
     },
     "message": "Successfully created checkout. Please complete payment at the invoice URL."
   }
   ```

## Service Fulfillment Filters

For certain proxy types, you can specify service fulfillment filters to customize the provisioning:

```json
"service_fulfillment_filter": {
  "asn_id": 7018,        // Specific ASN (e.g., AT&T)
  "subdivision_id": "us-tx",  // Specific region (e.g., Texas)
  "city_id": 75202       // Specific city (e.g., Dallas)
}
```

These filters allow you to target proxies with specific attributes such as:
- Network provider (ASN)
- Geographic location (country, subdivision, city)
- Protocol preferences (IPv4, IPv6, dual-stack)

## Managing Account Credit

Your account credit serves as a prepaid balance that can be applied to any purchase. This credit balance can come from:

1. **Direct top-ups**: Adding funds to your account
2. **Refunds**: Credits issued for service issues
3. **Promotional credits**: Special credits from promotional campaigns

### Viewing Credit Balance

Check your current credit balance:

```bash
curl --request GET \
  --url 'https://api.pingproxies.com/1.0/public/user/customer/retrieve' \
  --header 'X-API-Public-Key: your_public_key' \
  --header 'X-API-Private-Key: your_private_key'
```

The response includes your current credit balance:

```json
{
  "data": {
    "credit_balance": 500,
    // Other customer fields...
  },
  "message": "Customer successfully retrieved."
}
```

### Adding Credit

To top up your credit balance, use the checkout endpoint with the credit product:

```bash
curl --request POST \
  --url 'https://api.pingproxies.com/1.0/public/user/checkout/create' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Public-Key: your_public_key' \
  --header 'X-API-Private-Key: your_private_key' \
  --data '{
    "product_code": "credit",
    "quantity": 1,
    "cycle_interval": "month",
    "cycle_interval_count": 1,
    "top_up_amount": 5000
  }'
```

Response:

```json
{
  "data": {
    "invoice_is_paid": false,
    "invoice_url": "https://invoice.stripe.com/i/acct_1H7dAB2BUlqim5l/test_YWNjdF8xSGIzUEFCMkJabsEyzPS3Aag",
    "credit_added": true
  },
  "message": "Successfully created checkout. Please complete payment at the invoice URL."
}
```

## Managing Services

After purchasing proxies, you can manage your services through various API endpoints.

### Retrieving Service Details

```bash
curl --request GET \
  --url 'https://api.pingproxies.com/1.0/public/user/service/retrieve/API-1234-5678' \
  --header 'X-API-Public-Key: your_public_key' \
  --header 'X-API-Private-Key: your_private_key'
```

Response:

```json
{
  "data": {
    "service_id": "API-1234-5678",
```

### Editing a Service

You can modify certain aspects of an existing service:

```bash
curl --request PATCH \
  --url 'https://api.pingproxies.com/1.0/public/user/service/edit/API-1234-5678' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Public-Key: your_public_key' \
  --header 'X-API-Private-Key: your_private_key' \
  --data '{
    "service_is_automatic_collection": false,
    "service_metadata": {
      "project": "Client XYZ",
      "department": "Marketing"
    }
  }'
```

### Canceling a Service

```bash
curl --request DELETE \
  --url 'https://api.pingproxies.com/1.0/public/user/service/cancel/API-1234-5678' \
  --header 'Content-Type: application/json' \
  --header 'X-API-Public-Key: your_public_key' \
  --header 'X-API-Private-Key: your_private_key' \
  --data '{
    "cancel_feedback": "too_expensive",
    "cancel_comment": "Moving to a different solution"
  }'
```

## Residential Bandwidth Management

For residential proxies, which use a bandwidth-based model instead of a per-proxy model, you can monitor your bandwidth usage:

```bash
curl --request GET \
  --url 'https://api.pingproxies.com/1.0/public/user/residential/summary' \
  --header 'X-API-Public-Key: your_public_key' \
  --header 'X-API-Private-Key: your_private_key'
```

Response:

## Best Practices

1. **Test purchases with small quantities** before implementing large automated systems
2. **Monitor credit balance** to ensure it's sufficient for automated purchases
3. **Implement robust error handling** for checkout responses
4. **Track service IDs** returned from successful purchases
5. **Set up webhooks** (if available) to receive notifications about service status changes
6. **Maintain an internal inventory database** to track purchased proxies and their attributes

## API Limitations

Be aware of these limitations when automating purchases:

- Standard rate limits apply to all checkout endpoints
- A maximum of 100 proxies can be purchased in a single checkout
- Some purchase options may require manual verification for new accounts
- Certain geographic locations or network types may have limited availability
